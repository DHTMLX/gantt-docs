Что нового в MSP Project Export Module
===========================================

## 2.4.1.0

- Исправлена ​​ошибка, приводившая к сбою библиотеки MPXJ.Net при запуске модуля экспорта в контейнере Docker и импорте файлов MPP.

## 2.4.0.0

- Библиотека MPXJ перенесена из пакета net.sf.mpxj-for-csharp в пакет MPXJ.NET.
- Обновлена ​​библиотека MPXJ до версии 14.0.0.
- Расширен список поддерживаемых свойств проекта, задачи и ресурса для файлов Primavera P6.
- Добавлена ​​поддержка сущностей Primavera для импорта и экспорта:
Расходы проекта, Роли, Коды проекта, Коды действий, Коды ресурсов, Счета затрат.
- Добавлена ​​поддержка древовидной структуры для ресурсов.
- Добавлена ​​поддержка импорта нескольких базовых планов из файлов Primavera.
- Обновлены единицы измерения DurationFormat для более удобочитаемых значений.
- Добавлен способ экспорта задержки связи с пользовательским форматом длительности.
- Исправлен импорт файлов Primavera, содержащих пустые или нулевые календари.
- Добавлена ​​экспериментальная поддержка импорта файлов из других источников. Программы планирования (Asta, ProjectLibre и др.)

## 2.3.0.0

- Библиотеку MPXJ обновлена до версии 12.10.3.
- Добавлена поддержка стандартных типов задач (Project и Milestone) из коробки. Вехи в Primavera имеют тип `FINISH_MILESTONE`
- Добавлена поддержка Начальных и Финальных вех для экспорта Primavera
- Добавлена поддержка экспорта всех 11 базовых линий в файлы MSP
- Добавлена поддержка расширенных полей разных типов: Number1-20, Flag1-20, Cost1-10, Date1-10, Start1-10, Finish1-10, Duration1-10, OutlineCode1-10
- Добавлена поддержка использования точных имен для свойств Text1-30. Теперь это работает, когда вы указываете псевдоним ("My custom prop") или имя поля ("Text1")
- Расширен список поддерживаемых свойств задач в файлах MSP. Теперь они загружаются как стандартные свойства, а не как пользовательские/расширенные поля
- Добавлена поддержка стандартных свойств ресурсов и расширенных полей: Number1-20, Flag1-20, Cost1-10, Date1-10, Start1-10, Finish1-10, Duration1-10, OutlineCode1-10
- Добавлен способ импортировать большую часть стандартных свойств ресурсов одной настройкой свойства (чтобы не перечислять все свойства)
- Исправлен экспорт ресурса, когда назначения не имеют значения единиц
- Исправлено несколько ошибок, которые препятствовали импорту и экспорту файлов из-за некоторых значений в свойствах задач

## 2.2.1.0

- Исправлено указание значения в 24-часовом формате как конфигурации рабочего времени (например, `gantt.setWorkTime({ hours: ["00:00-24:00"] })`)
- Исправлено возвращаемое значение календаря проекта при указании свойства `CalendarUID` в параметре `projectProperties` функций импорта

## 2.2.0.0

- Библиотеку MPXJ обновлена до версии 12.0.0
- Скорректирован импорт свойств `Start` и `Finish`, указанных в параметре `taskProperties`
- Исправлен импорт при указании дубликат значений в параметре `taskProperties`
- Исправлен импорт при указании свойства `CalendarUID` в параметре `taskProperties` и если у задачи нет календаря
- Исправлен экспорт, когда `gantt.config.worktimes` содержит пустой массив
- Исправлен экспорт при указании настроек рабочего времени в старом формате

## 2.1.1.0

- Переименовано свойство, содержащее календари, с `worktimes` на `calendars`
- Добавлено свойство, содержащее идентификатор глобального календаря
- Восстановлено свойство `worktime`, содержащее настройки глобальных календарей для обратной совместимости

## 2.1.0.0

- Добавлена поддержка настраиваемых календарей для импортируемых и экспортируемых MSP и Primavera файлов (включая календарь ресурсов)
- Добавлена поддержка получения значений назначений ресурсов из единиц измерения при импорте файлов
- Добавлена поддержка указания пользовательских значений для единиц назначения ресурсов при экспорте файлов
- Добавлено свойство **RemainingDuration** для экспорта Primavera, чтобы получить ту же конечную дату задач без дополнительной конфигурации

## 2.0.2.0

- Обновлена библиотека MPXJ до версии 11.5.4
- Добавлена поддержка пользовательских свойств задач Summary (WBS) для импорта и экспорта Primavera. Это работает даже если свойства имеют одинаковое имя. Чтобы это работало для задач Summary при экспорте данных, необходимо вернуть true для свойства **Summary**
- Исправлена ошибка, когда базовая линия 0 не импортировалась

## 2.0.0.1

- Различные внутренние оптимизации и импортируемый Dockerfile

## 2.0.0.0

- Перенесено с ASP.NET MVC на ASP.NET Core. Теперь можно запускать модуль экспорта MSP на Linux и внутри Docker-образа
