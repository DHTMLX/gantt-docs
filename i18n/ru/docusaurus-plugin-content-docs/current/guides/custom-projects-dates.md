# Ручное планирование сводных задач

Есть возможность вручную планировать [проекты](guides/task-types.md) (сводные задачи).
Эта функция увеличивает гибкость и точность управления проектами с использованием диаграмм Ганта.

По умолчанию сводные задачи автоматически рассчитывают свои даты на основе самых ранних дат начала и самых поздних дат окончания их подзадач.
Также можно задать фиксированные даты начала и окончания для сводных задач независимо от их подзадач. В результате диаграмма Ганта будет отображать как фиксированную длительность, так и длительность, полученную от подзадач.

[Ручное планирование проектов](https://docs.dhtmlx.com/gantt/samples/04_customization/25_project_dates.html)

Чтобы активировать эту функцию для сводной задачи, установите свойство [auto_scheduling](api/config/auto_scheduling.md) в *false*.

Фиксированные даты хранятся в **task.start_date** и **task.end_date**, тогда как вычисляемые даты из подзадач доступны в **task.$auto_start_date** и **task.$auto_end_date**.

~~~js
gantt.parse({
  data: [
    {
      id: 1,
      text: "Project Phase 1",
      type: "project",
      start_date: "2025-05-01 00:00:00",
      duration: 15,
      auto_scheduling: false /* ! */
    },
    // ...
  ],
});
~~~

Как и фиксированная длительность сводной задачи, так и фактическая длительность, вычисленная по подзадачам, отображаются на диаграмме Ганта.

Если диапазон дат подзадачи выходит за пределы назначенных дат сводной задачи, сводная задача будет выделена, чтобы указать на конфликт планирования. Этот визуальный сигнал помогает конечным пользователям быстро выявлять и устранять несоответствия в графике проекта.
