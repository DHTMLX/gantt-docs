MSP 프로젝트 내보내기 모듈의 새로운 기능
===========================================

## 2.3.0.0

- MPXJ 라이브러리를 12.10.3 버전으로 업데이트했습니다.
- 표준 작업 유형(Project와 Milestone)을 기본적으로 지원합니다. Primavera의 Milestones는 `FINISH_MILESTONE` 타입을 가집니다.
- Primavera 내보내기에 Start 및 Finish Milestones 지원을 추가했습니다.
- MSP 파일로 11개의 Baselines를 모두 내보내는 지원을 추가했습니다.
- 확장 필드의 다양한 타입에 대한 지원을 추가했습니다: Number1-20, Flag1-20, Cost1-10, Date1-10, Start1-10, Finish1-10, Duration1-10, OutlineCode1-10
- Text1-30 속성에 대해 정확한 이름을 사용할 수 있는 지원을 추가했습니다. 이제 alias("My custom prop") 또는 필드 이름("Text1")을 지정해도 작동합니다.
- MSP 파일에서 지원되는 작업 속성 목록을 늘렸습니다. 이제 표준 속성으로 로드되며 커스텀/확장 필드로 로드되지 않습니다.
- 표준 리소스 속성과 확장 필드: Number1-20, Flag1-20, Cost1-10, Date1-10, Start1-10, Finish1-10, Duration1-10, OutlineCode1-10에 대한 지원을 추가했습니다.
- 거의 모든 표준 리소스 속성을 단일 속성으로 가져오는 방법을 추가했습니다(모든 속성을 나열할 필요가 없도록).
- 할당에 단위 값이 없을 때 리소스 내보내기를 수정했습니다.
- 작업 속성의 일부 값으로 인해 파일 가져오기 및 내보내기가 차단되던 다양한 오류를 수정했습니다.

## 2.2.1.0

- 작업 시간 구성에서 24시간 값을 지정하는 문제를 수정했습니다(예: `gantt.setWorkTime({ hours: ["00:00-24:00"] })`).
- 가져오기 함수에서 `projectProperties` 매개변수에 `CalendarUID` 속성을 지정할 때 반환된 프로젝트 달력을 수정했습니다.

## 2.2.0.0

- MPXJ 라이브러리를 12.0.0 버전으로 업데이트했습니다.
- `taskProperties` 매개변수에 지정된 Start 및 Finish 속성의 가져오기를 올바르게 처리하도록 수정했습니다.
- `taskProperties` 매개변수에서 중복 값을 지정할 때의 가져오기를 수정했습니다.
- `taskProperties` 매개변수에서 `CalendarUID` 속성을 지정하고 태스크에 달력이 없을 때의 가져오기를 수정했습니다.
- `gantt.config.worktimes`가 빈 배열일 때의 내보내기를 수정했습니다.
- 구식 형식으로 작동 시간 설정을 지정할 때의 내보내기를 수정했습니다.

## 2.1.1.0

- 달력을 포함하는 속성의 이름을 `worktimes`에서 `calendars`로 변경했습니다(``worktimes``에서 ``calendars``로 변경).
- 글로벌 달력의 ID를 포함하는 속성을 추가했습니다.
- 호환성을 위해 글로벌 달력 설정을 포함하는 `worktime` 속성을 복원했습니다.

## 2.1.0.0

- 가져오기 및 내보내기 MSP와 Primavera 파일용 커스텀 달력(리소스 달력 포함) 지원을 추가했습니다.
- 가져오기 시 단위에서 리소스 할당 값을 가져오는 지원을 추가했습니다.
- 내보내기 시 리소스 할당 단위에 대해 커스텀 값을 지정하는 지원을 추가했습니다.
- Primavera 내보내기에 RemainingDuration 속성을 추가하여 추가 구성 없이도 작업의 종료일이 같아지도록 했습니다.

## 2.0.2.0

- MPXJ 라이브러리를 11.5.4 버전으로 업데이트했습니다.
- Primavera 가져오기 및 내보내기를 위한 Summary (WBS) 작업의 사용자 정의 속성 지원을 추가했습니다. 속성 이름이 같아도 작동합니다. 데이터를 내보낼 때 Summary 작업에서 작동하게 하려면 **Summary** 속성에 대해 *true*를 반환해야 합니다.
- Baseline 0이 가져와지지 않던 버그를 수정했습니다.

## 2.0.0.1

- 다양한 내부 최적화 및 Dockerfile 가져오기를 적용했습니다.

## 2.0.0.0

- ASP.NET MVC에서 ASP.NET Core로 마이그레이션했습니다. 따라서 이제 MSP 내보내기 모듈을 Linux에서 및 Docker 이미지 안에서 실행할 수 있습니다.
