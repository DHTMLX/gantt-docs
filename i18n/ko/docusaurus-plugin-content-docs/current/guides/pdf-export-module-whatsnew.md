PDF Export Module의 새로운 기능
==============================

## 0.6.6

• 'visual' 설정을 사용하여 Excel로 내보낼 때, 일정이 없는 작업이 무시됩니다.

• DHTMLX Gantt 9.0용 템플릿이 추가되었습니다.

## 0.6.5

• DHTMLX Scheduler 7.0 템플릿이 조정되었습니다.

## 0.6.4

• 하나의 PDF 파일 내에서 멀티페이지 내보내기가 도입되었습니다(차트가 지정된 형식에 따라 청크로 내보내진 후, 페이지가 하나의 파일로 병합됩니다).

• 하나의 파일로 멀티페이지 내보내기 시 각 페이지에 그리드 및 타임라인 헤더를 표시하는 옵션이 추가되었습니다.

## 0.6.3

• Docker 이미지에서 Node.js 환경의 기본 이미지를 버전 20으로 업데이트했습니다.

• Dockerfile이 최적화되어 Docker 이미지 크기가 더 작아졌습니다.

## 0.6.2

• DHTMLX Scheduler 7.0용 템플릿이 추가되었습니다.

## 0.6.0

• MS Project / Primavera P6 내보내기 서비스의 엔드포인트가 변경되었습니다.

• "visual:true" 파라미터 사용 시 Excel 내보내기에서 타임라인 셀이 올바르게 강조 표시되도록 수정되었습니다.

• Docker 파일 빌드에 사용되는 패키지가 업데이트되었습니다.

## 0.5.9

• Docker 이미지에서 Node.js 버전이 업그레이드되었습니다.

• 보안 경고 해결을 위해 Docker 이미지에서 사용되지 않는 파일이 제거되었습니다.

• 그리드 열에 포매터가 있을 때 내보내기가 실패하는 이슈가 수정되었습니다.

## 0.5.8

• 내보내기 모듈의 메인 페이지가 추가되었으며, 기능 테스트를 위한 링크가 제공됩니다(테스트 전 드롭다운의 URL을 반드시 확인하세요).
예시: [https://export.dhtmlx.com](https://export.dhtmlx.com)

• 새로운 기능이 도입되었습니다: [Node.js용 Gantt의 가져오기 및 내보내기](guides/export-nodejs.md).

• `visual:true` 파라미터와 관련된 Excel 내보내기 버그가 수정되어, 여러 스케일, 다양한 기간 단위, 타임라인 셀 경계에 정확히 맞지 않는 작업도 내보낼 수 있습니다.

## 0.5.7

• Node.js 14와의 호환성이 향상되었습니다.

• 커스텀 로케일 관련 이슈가 수정되었습니다.

• Docker용 "init" 프로세스가 추가되었습니다.

## 0.5.6

• 헤드리스 서버(그래픽 인터페이스 없이 실행) 및 Docker 내에서 발생하는 메모리 누수가 해결되었습니다.

## 0.5.5

• Docker 이미지 내 애플리케이션의 전체 성능이 향상되었습니다.

## 0.5.0

• PDF 및 PNG 내보내기를 위해 PhantomJS가 Electron으로 대체되었습니다.
